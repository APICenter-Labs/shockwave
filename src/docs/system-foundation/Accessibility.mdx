  ## Core expectations

  - Use semantic markup before adding custom interaction layers.
  - Preserve keyboard navigation and logical tab order.
  - Keep readable contrast between text, backgrounds, and interactive states.
  - Provide labels, text alternatives, and descriptive content where components need them.
  - Treat hover, focus, active, and disabled states as part of the design contract.

  <Note title="Current foundation status">
    Shockwave is still early, so this page establishes the baseline rules that future components should follow as the design system expands.

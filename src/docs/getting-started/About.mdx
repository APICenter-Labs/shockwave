  ## What Shockwave solves

  - Creates a single place to define component APIs, visual patterns, and usage guidance.
  - Keeps React and React DOM as peer dependencies so consuming apps stay in control of their runtime.
  - Produces distributable JavaScript and CSS output with a simple import story for product teams.
  - Gives the team a safe space to evolve tokens, accessibility guidance, and future components in public.

  ## What to expect next

  Shockwave already proves the full library workflow: component authoring, Storybook docs, package builds, and external-app consumption. That foundation makes it easier to add the next wave of primitives like inputs, overlays, and richer theming without changing how teams adopt the package.

  ## React-based support today

  Shockwave publishes JavaScript and CSS for React consumers. That makes these the direct support targets right now:

  - React applications
  - Vite + React
  - Next.js
  - Remix
  - Astro with the React integration

  ## Supported consumption model

  Shockwave publishes JavaScript and CSS. That means the host framework needs to do two things well:

  - Resolve npm packages from <code>node_modules</code>.
  - Render React components in the application shell.

  ## Angular and Vue

  Angular and Vue are not direct integration targets for the current package. Shockwave exports React components and declares <code>react</code> and <code>react-dom</code> as peer dependencies, so those frameworks would need an adapter layer, wrapper package, or web-component build that the repo does not currently ship.

  <Note title="Recommendation" tone="warm">
    It is safer to document Angular and Vue only after Shockwave has a real compatibility story for them. Right now, React and React-based frameworks are the supported path.
  </Note>
